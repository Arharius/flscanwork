Ты — опытный Full-Stack разработчик, специализирующийся на автоматизации и интеграции API. Твоя задача — создать полнофункциональное приложение для автоматического поиска и обработки фриланс-заказов.

**Основные требования:**

Приложение должно работать как автономный сервис 24/7 без необходимости ручного вмешательства.

**Функциональность поиска:**
- Мониторить основные фриланс-биржи (Upwork, Fiverr, Freelancer.com, PeoplePerHour и локальные платформы РФ: Kwork, Weblancer, Fl.ru, Advego, YouDo)
- Искать заказы, которые можно реализовать с помощью Viber-бота (веб-кодирование, автоматизация сообщений, интеграция с Viber API, создание простых ботов на Python/Node.js, webhook-интеграции)
- Фильтровать по релевантности: язык программирования, сложность, бюджет, сроки
- Исключать уже обработанные заказы и откликнутые предложения

**Функциональность откликов:**
- Анализировать описание каждого заказа и извлекать ключевую информацию (требования, технологии, бюджет, сроки)
- Генерировать персонализированные отклики, которые:
  - Демонстрируют понимание конкретной задачи
  - Содержат релевантные примеры/кейсы работы с Viber-ботами
  - Звучат профессионально и естественно (не как автоматический ответ)
  - Соответствуют тону и стилю заказчика
  - Указывают конкретные технологии и подход к решению

**Технические требования:**
- Использовать готовые API фриланс-бирж (где доступны) или web-scraping с соблюдением robots.txt
- Использовать LLM (GPT, Claude или аналог) для генерации персонализированных текстов
- Реализовать базу данных для отслеживания обработанных заказов и статистики откликов
- Логирование всех действий (найденные заказы, отправленные отклики, ошибки)
- Возможность настройки параметров (ключевые слова, минимальный бюджет, время работы)

**Архитектура:**
- Асинхронный сервис с использованием `asyncio` и `AsyncIOScheduler` (APScheduler)
- Периодический запуск поиска (каждые 15-30 минут, настраивается)
- Модульная структура с `BasePlatform` для унификации работы с разными биржами (поиск → анализ → генерация отклика → отправка)
- Обработка ошибок и повторные попытки
- Уведомления о результатах (лог-файлы, опционально — email/Telegram)

**Языки и технологии:**
- Python 3 (основной язык) с асинхронной работой через `asyncio`
- `httpx` (версия 0.27.0) для HTTP-запросов к API бирж
- `python-dotenv` (версия 1.0.0) для управления переменными окружения (API-ключи, конфигурация)
- Selenium/Playwright для web-scraping (если нужно)
- База данных: SQLite или PostgreSQL
- Scheduling: APScheduler (версия 3.10.4)
- Управление зависимостями: `pip install -r requirements.txt`

**Входные данные:**
Используй информацию из приложенного файла `usrbinenv python3.txt`, который содержит базовую структуру асинхронного сервиса с примерами поиска заказов, генерации откликов и работы с уведомлениями. Файл `deepseek_toml_20260418_cec26a.txt` определяет конфигурацию окружения и команду для запуска приложения (`python main.py`).

Создай полный код приложения, включая:
- Все необходимые модули (поиск по биржам, анализ заказов, генерация откликов, управление БД)
- Конфигурацию через переменные окружения
- Инструкции по установке и запуску
- Примеры использования
- Логирование и обработку ошибок

Приложение должно быть готово к запуску на VPS или локальной машине без дополнительной ручной настройки.
