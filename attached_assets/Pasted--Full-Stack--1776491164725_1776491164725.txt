Ты — опытный Full-Stack разработчик, специализирующийся на системах машинного обучения, аналитике данных и оптимизации. Твоя задача — создать систему обратной связи и continuous optimization для существующего приложения поиска фриланс-заказов, которое уже генерирует персонализированные отклики.

**Основная цель:**
Построить замкнутый цикл обучения, который отслеживает результаты каждого отправленного отклика, анализирует корреляции между характеристиками текста и успехом, и использует эти данные для постоянного улучшения качества генерируемых откликов через fine-tuning LLM-запросов и адаптацию стиля.

**Ключевые компоненты системы:**

1. **Модуль отслеживания метрик**
   - Перехватывать и записывать каждый отправленный отклик с полным текстом, временной меткой и ID заказа
   - Отслеживать три типа результатов: получен ответ заказчика, приглашение на проект, отклонение
   - Синхронизировать данные с платформами через API (где доступно) или ручной логирование пользователем
   - Хранить статус каждого отклика с датой и временем получения результата

2. **Модуль анализа корреляций**
   - Анализировать взаимосвязь между успешностью и параметрами отклика: длина текста, частота использованных ключевых слов, количество примеров кейсов, тон (formal/casual), наличие цены/сроков в отклике
   - Вычислять коэффициент успешности (success rate) для каждого параметра по категориям заказов
   - Выявлять статистически значимые закономерности (например, "отклики длиной 150-200 слов получают на 35% больше ответов для проектов Python")
   - Выделять антипаттерны (характеристики, которые коррелируют с отклонениями)

3. **Модуль хранения и историзации**
   - Сохранять в базе данных каждый отклик с полным текстом, характеристиками заказа (технологии, бюджет, сроки, язык), результатом и метаданными
   - Маркировать отклики как "успешные" или "неудачные" на основе отслеживаемых результатов
   - Создавать индексированный архив для быстрого поиска и анализа по категориям
   - Версионировать — сохранять информацию о том, какая версия LLM-промпта была использована при генерации отклика

4. **Модуль адаптивной генерации откликов**
   - На основе анализа успешных паттернов динамически корректировать system prompt для LLM
   - Встраивать выявленные закономерности в контекст: "Для заказов Python без указания бюджета отклики длиной 180-200 слов с 2-3 примерами кейсов получают на 40% больше ответов"
   - Использовать метрики успешности для динамического выбора стиля (формальный, дружелюбный, технический) в зависимости от требований заказчика
   - Экспериментировать с вариантами через A/B тестирование (генерировать 2-3 варианта отклика и выбирать наиболее перспективный на основе исторических данных)

5. **Модуль распознавания паттернов заказчиков**
   - Автоматически извлекать и кластеризировать требования: язык программирования, фреймворк, бюджет-диапазон, сроки (срочные, стандартные, гибкие), тип проекта (новая разработка, поддержка, интеграция)
   - Отслеживать популярные сочетания параметров (например, "Python + интеграция Viber + бюджет $100-500")
   - Адаптировать стиль отклика в зависимости от профиля заказчика: длину текста, уровень технического детализма, наличие/отсутствие примеров
   - Автоматически обновлять "портреты" успешных заказчиков на основе проведенных проектов

6. **Модуль еженедельной аналитики и отчетов**
   - Генерировать еженедельные отчеты с таблицами коэффициентов успешности по категориям (по языкам, по типам проектов, по бюджет-диапазонам)
   - Включать тренды: улучшилась ли эффективность за неделю, какие характеристики откликов показали лучший результат
   - Подчеркивать actionable insights: "На этой неделе отклики без указания конкретного подхода получали ответы в 25% случаев vs 60% для откликов с детальным техническим планом"
   - Давать рекомендации по оптимизации (например, "рекомендуем снизить целевую длину откликов для заказов с бюджетом >$1000")

**Технические требования:**

- **База данных:** PostgreSQL с таблицами для откликов (response_id, text, characteristics, timestamp), результатов (result_id, response_id, status, timestamp), параметров заказов (job_id, languages, budget, deadline)
- **Анализ данных:** Python с pandas, numpy, scipy для статистических расчетов и выявления корреляций
- **LLM интеграция:** Использовать API Claude/GPT с динамически генерируемыми system prompts на основе выявленных паттернов
- **Scheduling:** APScheduler для еженедельного запуска аналитики и генерации отчетов
- **Экспорт отчетов:** JSON, CSV, опционально — HTML-таблицы или отправка через email/Telegram

**Выходные данные:**

1. Модульная архитектура системы с четким интерфейсом между компонентами
2. Полный код всех модулей (tracking, analysis, storage, generation, reporting)
3. Schema базы данных с индексами для производительности
4. Примеры еженедельных отчетов и визуализаций
5. Инструкция по интеграции с существующим приложением поиска и генерации откликов
6. Документация по настройке порогов анализа и параметров A/B тестирования

**Цель оптимизации:** Система должна работать автономно 24/7, непрерывно собирая данные о результативности откликов и автоматически улучшая качество генерируемых текстов. Каждый цикл обучения (неделя) должен показывать измеримый прирост коэффициента успешности благодаря применению выявленных паттернов.
